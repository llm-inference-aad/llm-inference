# 01 - Environment Setup & Installation

This guide covers how to set up the project environment using `uv`, the recommended high-performance Python package manager.

## 1. Install `uv`

If `uv` is not already installed on the system, you can install it into your user directory without requiring admin privileges.

```bash
# This script installs uv to ~/.local/bin
curl -LsSf https://astral.sh/uv/install.sh | sh

# Ensure ~/.local/bin is in your PATH
export PATH="$HOME/.local/bin:$PATH"
```

## 2. Create the Virtual Environment

The project is tested and compatible with **Python 3.12**.

1.  **Create the venv**: This command creates a new virtual environment in the `.venv` directory.
    ```bash
    uv venv .venv --python 3.12
    ```

2.  **Activate the venv**:
    ```bash
    source .venv/bin/activate
    ```

## 3. Install Dependencies

`uv sync` reads the `pyproject.toml` file and installs all required dependencies. It's significantly faster than `pip install`.

```bash
# Ensure the virtual environment is active
uv sync
```
This process typically completes in under 3 minutes and installs over 100 packages, including PyTorch, Transformers, and DEAP.

### Optional: vLLM Backend (Continuous Batching)

For higher throughput, you can enable the vLLM inference backend:

```bash
uv sync --extra vllm
```

Then set `USE_VLLM=true` in your `.env` file. The server will use vLLM's continuous batching for better performance during LLM Guided Evolution.

**Note**: vLLM has additional build dependencies (e.g., compatible CUDA, possibly Rust for some transitive deps). If installation fails, the server automatically falls back to the HuggingFace backend when `USE_VLLM=true` but vLLM is not installed.

## 4. Using `uv run`

As an alternative to activating the environment, you can use `uv run` to execute commands within the managed environment. This is useful for one-off script executions.

```bash
# No need to run 'source .venv/bin/activate'
uv run python scripts/plot_latency_vs_accuracy.py --help
```

## 5. Key Files

- **`pyproject.toml`**: Defines all project dependencies. This is the source of truth for `uv sync`.
- **`uv.lock`**: A lock file generated by `uv` for deterministic, reproducible builds. This file is not tracked in git.
- **`.gitignore`**: Configured to ignore `.venv/`, `uv.lock`, and any backups (`.venv-backup*/`).
